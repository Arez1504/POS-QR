=============================================================================
             HƯỚNG DẪN THƯ MỤC & TẬP TIN - HỆ THỐNG POS QR-CODE
=============================================================================

Tài liệu này ghi chú toàn bộ cấu trúc thư mục, chức năng và ý nghĩa của từng tập tin trong dự án POS Bán hàng kết hợp quét mã vạch/QR qua Điện thoại & Camera.

-----------------------------------------------------------------------------
1. THƯ MỤC GỐC (ROOT)
-----------------------------------------------------------------------------
* [SETUP_GUIDE.md]           : Tài liệu hướng dẫn cài đặt nhanh dự án (cách chạy database, backend, frontend).
* [huongdan.txt]             : Tài liệu này (sơ đồ tổ chức các thư mục và tập tin trong dự án).

-----------------------------------------------------------------------------
2. THƯ MỤC CƠ SỞ DỮ LIỆU (database/)
-----------------------------------------------------------------------------
Thư mục chứa các script SQL dùng để khởi tạo và nâng cấp cơ sở dữ liệu MySQL:
* [schema.sql]               : Bản thiết kế chính của cơ sở dữ liệu (tạo database 'pos_qr', bảng users, products, orders, order_items, shifts...). Chứa luôn câu lệnh insert tài khoản admin mặc định.
* [migration_v2_customers.sql]: Script nâng cấp thêm bảng 'customers' để quản lý thành viên, tích lũy điểm và giao dịch điểm (loyalty).
* [migration_v3_activity_logs.sql]: Script nâng cấp thêm bảng 'activity_logs' để lưu vết hành động của người dùng (audit log).
* [migration_v4_ai_chat.sql]: Script nâng cấp thêm bảng 'ai_chat_history' để lưu lịch sử hội thoại của người dùng với Trợ lý AI.
* [workbench_queries.sql]     : Tổng hợp các câu lệnh SQL mẫu để test, truy vấn nhanh trên MySQL Workbench (xem doanh thu, xem tồn kho, xem lịch sử ca...).

-----------------------------------------------------------------------------
3. THƯ MỤC BACKEND (backend/) - Node.js Express API
-----------------------------------------------------------------------------
Thư mục chịu trách nhiệm xử lý API logic, tính toán, kết nối cơ sở dữ liệu và trung chuyển Socket.IO:
* [.env]                     : Cấu hình biến môi trường (Thông tin kết nối MySQL, JWT Secret Key, Cổng chạy server...).
* [server.js]                : Điểm khởi chạy của Backend. Thiết lập HTTP Server, Socket.IO Server (để đồng bộ quét mã từ điện thoại), kết nối các routes API chính và chạy Server.
* [src/config/db.js]         : Khởi tạo Connection Pool kết nối MySQL bằng thư viện 'mysql2/promise'.
* [src/config/aiConfig.js]   : Cấu hình mô hình AI (chỉ định model name, API key của Gemini và system instruction prompt).
* [src/middleware/authMiddleware.js]: Chứa các hàm middleware kiểm tra token đăng nhập (JWT) và phân quyền truy cập theo vai trò (admin, cashier, warehouse).

* [src/controllers/] - Xử lý logic API (giao tiếp với DB và trả về JSON):
  - [activityController.js]  : Truy vấn danh sách lịch sử hoạt động hệ thống.
  - [aiController.js]        : Xử lý logic hội thoại với Gemini, lưu trữ lịch sử chat vào DB, trả về câu hỏi gợi ý.
  - [aiTools.js]             : Định nghĩa danh sách các tool và hàm thực thi truy vấn DB MySQL (READ-ONLY) để Gemini tự gọi (Function Calling).
  - [authController.js]      : Xử lý đăng ký, đăng nhập và lấy thông tin người dùng hiện tại.
  - [customerController.js]  : CRUD khách hàng thành viên, tìm kiếm theo số điện thoại.
  - [inventoryController.js] : Quản lý nhập hàng vào kho và nhật ký xuất nhập tồn (inventory logs).
  - [orderController.js]     : Xử lý tạo đơn hàng mới, tính toán điểm tích lũy/tiêu điểm và lưu thông tin chi tiết hóa đơn.
  - [paymentController.js]   : Giả lập và kiểm tra trạng thái thanh toán chuyển khoản bằng VietQR.
  - [productController.js]   : Xử lý CRUD sản phẩm, danh mục sản phẩm.
  - [qrController.js]        : Tạo dữ liệu mã vạch và QR hỗ trợ in ấn.
  - [reportsController.js]   : Tính toán thống kê doanh thu, biểu đồ tài chính, hiệu suất thu ngân.
  - [settingController.js]   : Quản lý thiết lập của cửa hàng (tên quán, cấu hình ngân hàng VietQR).
  - [shiftController.js]     : Quản lý ca làm việc (Mở ca, đối soát két tiền thực tế khi đóng ca).
  - [userController.js]      : Quản lý danh sách nhân viên (Thêm/Sửa/Xóa tài khoản).

* [src/routes/] - Định nghĩa URL các API Endpoints:
  - Các tệp tin *.js tương ứng với từng nhóm nghiệp vụ như: `/api/auth`, `/api/products`, `/api/orders`, `/api/shifts`, `/api/customers`, `/api/inventory`, `/api/ai`... Chúng kết nối các API URL với controller xử lý tương ứng và áp dụng các middleware bảo mật.

* [src/utils/logger.js]      : Hàm tiện ích ghi nhận tự động các hoạt động quan trọng (Thêm sản phẩm, Đóng ca, Đăng xuất...) vào bảng `activity_logs`.

=============================================================================
5. CÁC THUẬT TOÁN & PHƯƠNG PHÁP XỬ LÝ CHÍNH
=============================================================================

Hệ thống áp dụng một số thuật toán và quy trình xử lý dữ liệu đặc trưng dưới đây:

-----------------------------------------------------------------------------
A. Mã Hóa Mật Khẩu (Bcrypt Password Hashing)
-----------------------------------------------------------------------------
* Thuật toán: Sử dụng thuật toán băm Bcrypt (kết hợp Salt) để băm mật khẩu người dùng trước khi lưu vào DB (trong authController.js).
* Tác dụng: Ngăn ngừa lộ mật khẩu ngay cả khi cơ sở dữ liệu bị truy cập trái phép.

-----------------------------------------------------------------------------
B. Xác Thực Phân Quyền Không Trạng Thái (Stateless JWT Authentication)
-----------------------------------------------------------------------------
* Thuật toán: Sử dụng thuật toán chữ ký HS256 (HMAC SHA256) của JSON Web Token để ký số thông tin phiên làm việc.
* Tác dụng: Xác thực quyền hạn người dùng (admin, cashier) trên từng request API mà không cần lưu session ở backend.

-----------------------------------------------------------------------------
C. Tìm Kiếm Mờ & Khớp Dữ Liệu (Fuzzy Search & Substring Matching)
-----------------------------------------------------------------------------
* Thuật toán: 
  - Backend sử dụng truy vấn SQL LIKE `%keyword%` kết hợp để tìm kiếm trên nhiều trường cùng lúc (Tên, SKU, Mã vạch).
  - Frontend sử dụng hàm khớp chuỗi con `includes()` trong JS để tìm kiếm và khớp nhanh sản phẩm ngay trên giao diện POSPage khi gõ từ khóa.

-----------------------------------------------------------------------------
D. Quy Trình Tích Điểm & Trừ Điểm Thành Viên (Loyalty Reward Point Logic)
-----------------------------------------------------------------------------
* Thuật toán:
  - Tích lũy: Điểm cộng thêm = `Math.floor(Tổng tiền đơn hàng / VND_PER_POINT)`.
  - Tiêu điểm: Giảm trừ tối đa 50% giá trị đơn hàng. Số tiền giảm = `Số điểm tiêu * VND_PER_POINT`.
  - Kiểm tra tính nhất quán: Hệ thống tính toán đối chiếu tiền mặt trả khách, tổng tiền, điểm tiêu dùng và điểm tích lũy trên cả client & server để tránh gian lận.

-----------------------------------------------------------------------------
E. Tra Cứu Trình Tự Bậc Thang (Cascaded API Lookup Fallback)
-----------------------------------------------------------------------------
* Thuật toán: Khi quét mã vạch không có sẵn, hệ thống thực hiện gọi tuần tự (Cascaded Fetch):
  1. Thử gọi API Open Food Facts (Pháp) trước.
  2. Nếu không tìm thấy, tự động chuyển sang gọi thử API UPCitemdb.
  3. Nếu cả hai thất bại hoặc offline, dừng luồng loading và chuyển về form trống cho phép người dùng tự gõ thông tin sản phẩm bằng tay.

-----------------------------------------------------------------------------
F. Đồng Bộ Thời Gian Thực Qua WebSockets (Real-time Socket Rooms)
-----------------------------------------------------------------------------
* Thuật toán: Thuật toán chia nhóm và ghép đôi (Pairing Room):
  - POS và ĐT quét cùng join vào một phòng (Room) có mã đặt tên theo `userId`.
  - Khi ĐT bắn sự kiện `barcode_scanned` kèm mã vạch, server Socket.IO lập tức định tuyến phát lại (broadcast) mã vạch đến đúng POS Room đó thông qua sự kiện `new_barcode`.

-----------------------------------------------------------------------------
G. Tạo Chuỗi Mã QR Thanh Toán Chuẩn VietQR (VietQR Payload Builder)
-----------------------------------------------------------------------------
* Thuật toán: Thiết lập chuỗi truy vấn động theo định dạng VietQR:
  - Ghép nối thông tin: Mã ngân hàng + Số tài khoản + Số tiền đơn hàng + Nội dung chuyển khoản (Mã đơn hàng) + Tên tài khoản nhận.
  - Tác dụng: Cho phép khách quét mã tự điền chính xác 100% số tiền và nội dung, giúp kiểm tra tự động (automated reconciliation).

-----------------------------------------------------------------------------
H. Kiểm Tra Trạng Thái Thanh Toán Tuần Hoàn (Reconciliation Polling)
-----------------------------------------------------------------------------
* Thuật toán: Polling lặp lại định kỳ (độ phức tạp O(1) mỗi chu kỳ):
  - Thiết lập một chu kỳ `setInterval` cứ 2.5 giây truy vấn trạng thái hóa đơn từ database.
  - Khi trạng thái thanh toán chuyển thành `paid`, lập tức dừng bộ đếm thời gian (`clearInterval`), tắt Modal thanh toán QR và hiển thị thông báo thành công.

-----------------------------------------------------------------------------
I. Đối Soát Két Tiền Mặt Cuối Ca (Cash Register Reconciliation)
-----------------------------------------------------------------------------
* Thuật toán: Tính toán lượng tiền mặt dự kiến:
  - `Tiền mặt dự kiến = Tiền mặt đầu ca (opening_cash) + Tổng doanh thu tiền mặt phát sinh trong ca`.
  - So sánh với `Tiền mặt thực tế đóng ca (closing_cash)` nhập bởi cashier để xác định khoản chênh lệch thừa/thiếu tiền két (discrepancy).

-----------------------------------------------------------------------------
K. Trợ Lý AI Tích Hợp Hệ Thống POS (AI Assistant with Function Calling)
-----------------------------------------------------------------------------
* Thuật toán: Chatbot AI sử dụng Google Gemini (gemini-2.5-flash) kết hợp gọi hàm (Function Calling):
  - Nhận tin nhắn từ người dùng và lịch sử cuộc trò chuyện (được tự động làm sạch tin nhắn trùng lặp ở backend).
  - Sử dụng các định nghĩa hàm (tools) để tự động nhận biết khi nào cần thông tin và thực thi các câu lệnh truy xuất trực tiếp từ cơ sở dữ liệu MySQL (doanh thu, tồn kho, sản phẩm bán chạy, hiệu suất ca làm việc) dưới dạng READ-ONLY.
  - Tự động dịch mã lựa chọn theo số thứ tự (ví dụ: người dùng chọn "1", hệ thống tự động ánh xạ sang tham số tương ứng để gọi tool phù hợp) mà không cần yêu cầu người dùng gõ lại toàn bộ nội dung.
  - Định dạng phản hồi: Tuyệt đối không dùng bảng (table) mà hiển thị dạng danh sách gạch đầu dòng hoặc đánh số rõ ràng, đồng thời không sử dụng bất kỳ biểu tượng cảm xúc (emoji/icon) nào trong câu trả lời.
